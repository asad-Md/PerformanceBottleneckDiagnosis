# Chapter 2: LITERATURE SURVEY

## 2.1 Introduction This chapter provides a comprehensive review of existing literature related to performance bottleneck diagnosis in computing systems. As modern architectures, especially multi-core and cloud-native environments, become increasingly complex, the need for efficient, low-overhead, and accurate performance profiling and diagnosis tools has grown significantly. This survey explores various techniques ranging from traditional performance profiling tools and advanced eBPF tracing to modern AI-driven diagnosis systems. By analyzing these methodologies, we identify the current state-of-the-art, highlight their strengths and weaknesses, and establish the research gaps that this project aims to address.

## 2.2 Review of Existing Techniques

### eBPF Tracing Extended Berkeley Packet Filter (eBPF) has emerged as a revolutionary technology for system observability, allowing for the execution of sandboxed programs within the kernel without modifying kernel source code or loading loadable kernel modules (LKMs) [12]. eBPF provides near-native execution speed with mathematically guaranteed kernel safety, eliminating the severe performance penalty of context switches by aggregating and filtering data directly inside the kernel [12]. Recent works have leveraged eBPF for zero-instrumentation tracing of machine learning systems, capturing multidimensional process-level metrics with near-zero monitoring overhead [15]. Furthermore, eBPF tracepoints have been dynamically implemented to track system call sequences for real-time security and anomaly detection, achieving ultra-low overhead (under 1%) and sub-millisecond latency [16].

### Performance Profiling Tools Traditional performance profiling often relies on hardware performance counters (HPCs) to gather low-level CPU metrics. Studies have shown that massive HPC data collection can be simplified through dimensionality reduction, revealing that a minimal set of highly specific hardware events is sufficient to predict system performance and even classify complex behaviors like ransomware execution on bare-metal systems [4], [5]. In parallel computing, detailed execution timeline analysis using tools like the Linux Trace Toolkit Next Generation (LTTng) helps construct Waiting Dependency Graphs (WDGs) to localize bottlenecks and analyze off-CPU time [9]. However, traditional unicore metrics are often misleading in multicore environments due to hidden shared-resource contention, necessitating multicore-aware profiling and optimization [11]. The deployment of loopback traffic tests directly between RDMA NICs and local CPU endpoints has also proven effective in precisely isolating intra-host hardware bottlenecks from standard network issues [10].

### AI-based Diagnosis Systems The integration of Machine Learning (ML) and Artificial Intelligence (AI) has significantly enhanced automated performance diagnosis. Advanced frameworks utilize unsupervised weighted spectrum analysis and graph networks on Service Call Graphs (SCGs) to accurately identify dual root causes in microservices [1]. Spatial-Temporal Dynamic Graph Neural Networks (DGNNs) coupled with Variational Auto-Encoders (VAEs) have been deployed to learn the evolving behavior of parallel programs, distinguishing between endogenous code flaws and exogenous noise [2]. Operation-centric dependency graphs with customized PageRank algorithms improve precision in pinpointing exact root cause locations at the operation level [3]. Other approaches include decentralized, trigger-based ML architectures using Multi-Layer Perceptrons (MLPs) at the edge to reduce telemetry bandwidth [6], and the application of Gaussian Mixture Models (GMMs) on eBPF data to cluster and classify hardware failures in distributed nodes [15].

## 2.4 Research Gap

Based on the extensive review of the literature, several critical research gaps have been identified in the current state of performance bottleneck diagnosis:

* **Real-time Limitations**: Many existing high-fidelity diagnostic tools rely on post-mortem analysis of massive trace datasets (such as MPI traces or full dependency graphs), making instantaneous, real-time bottleneck diagnosis highly challenging [2], [11]. Architectures that require heavy preprocessing or multi-pass profiling cannot be seamlessly integrated into live production environments [7]. * **Overhead Issues**: Traditional tracing methodologies, such as LTTng or full distributed tracing, often introduce significant Observer Effect overhead and data volume bloat [1], [9]. Capturing excessive execution context results in high storage costs and severe system distortion, which is unacceptable for continuous monitoring in sensitive backend systems. * **Scalability Challenges**: While some distributed architectures mitigate bandwidth issues, many ML models utilized in the literature (e.g., DGNNs, GMMs, and DRL) are either too complex or highly specific to certain domains (like 5G networks or GPU workloads), lacking the scalability and adaptability required for general-purpose, dynamic multi-core CPU environments [6], [15].

Furthermore, there is a distinct absence of minimalist architectures that seamlessly combine eBPF-driven telemetry with highly optimized AI models. Current solutions often suffer from misaligned ML goals—predicting benchmark scores or network telemetry rather than directly classifying localized multi-core root causes—and lack a streamlined eBPF-to-ML pipeline that restricts data collection to a purely minimal set of key performance indicators (KPIs) for direct neural network classification.

## 2.5 Chapter Summary This chapter surveyed a broad spectrum of techniques for performance bottleneck diagnosis, ranging from foundational performance profiling and advanced eBPF tracing to sophisticated AI-based detection systems. The comparative analysis of 15 key methodologies highlighted substantial advancements in root cause identification and anomaly detection across various computing environments. However, it also exposed significant research gaps, particularly concerning the high overhead of existing tracing tools, the lack of real-time multi-core diagnostic capabilities, and the absence of minimalist, AI-driven eBPF pipelines. Addressing these limitations is essential for developing a lightweight, accurate, and real-time bottleneck diagnosis framework, which forms the core motivation for this research project.
